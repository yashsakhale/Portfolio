Cloud AtlasClassifier

Project Motivation & Problem Statement

Emergency management systems ingest thousands of alerts daily from news feeds, government agencies, and IoT sensors—each needing rapid, accurate categorization to route to appropriate responders. Manual classification cannot keep pace, and traditional ML deployments require dedicated servers with complex scaling logic. Cloud AtlasClassifier solves this by building a hybrid serverless classification pipeline that combines AWS Bedrock's Claude 3.5 Haiku LLM with a custom-trained SageMaker multi-label classifier, achieving high accuracy with cost-efficient auto-scaling.

The core challenge: How do we classify alerts into 64+ granular categories (fires, earthquakes, infrastructure failures, cyberattacks, crimes) with production-grade reliability, sub-second latency, and zero infrastructure management?

Technical Approach

1. Event-Driven Serverless Architecture

  • Built an SQS-to-Lambda-to-SQS pipeline: alerts arrive in qIntUncategorized, trigger Lambda classification, and output to qIntCategorized with category assignments.
  • Defined the entire infrastructure in template.yml using AWS SAM, enabling one-command deployment (sam build --use-container && sam deploy --guided).
  • Configured Lambda with 512 MB memory, 90-second timeout, and X-Ray tracing for observability.
  • Used ReportBatchItemFailures for SQS event source mapping to enable partial batch retries on transient errors.

2. Hybrid SageMaker + Bedrock Classification

  • Deployed a custom multi-label classifier on SageMaker (categorizer-endpoint) using PyTorch and HuggingFace AutoModelForSequenceClassification fine-tuned on alert data.
  • Implemented confidence-based routing: SageMaker processes alerts first; if confidence < 0.8, the system falls back to Bedrock's Claude 3.5 Haiku (us.anthropic.claude-3-5-haiku-20241022-v1:0) for chain-of-thought reasoning.
  • Training pipeline includes AdamW optimizer with linear warmup schedule, early stopping, and model checkpointing for best validation loss.

3. Multi-Pass LLM Classification with Chain-of-Thought

  • Pass 1 (Title-Only): Uses build_hierarchical_prompt_title_only() for fast initial classification when context is sufficient.
  • Pass 2 (With Content): Falls back to build_hierarchical_prompt_with_content() with up to 2000 characters of article body for ambiguous cases.
  • Prompts include few-shot examples with explicit reasoning chains and comprehensive routing rules covering all 64 categories across domains: FLASH, ENVIRONMENT, GEOPOLITICS, INFRASTRUCTURE, UTILITIES, CYBERSECURITY, SUPPLY-CHAIN, CRIME.
  • Bedrock Converse API returns strictly validated JSON parsed via JSONDecoder.raw_decode() to handle markdown fences and preamble text.

4. Category Validation & Quality Scoring

  • CategoryValidator enforces business logic: detects conflicting pairs (e.g., CRIME + UTILITIES rarely co-occur) and validates complementary pairs (e.g., ENVIRONMENT_FIRES commonly pairs with INFRASTRUCTURE_ROADS-AND-TRAFFIC).
  • Strong keyword matching validates category assignments against expected terms (highway/interstate → ROADS-AND-TRAFFIC, subway/metro → MASS-TRANSIT, etc.).
  • Metrics instrumentation tracks pass1_success, pass2_needed, sagemaker_confident, sagemaker_fallback_to_bedrock rates for continuous improvement.

5. SageMaker Training Pipeline

  • train.py implements JsonlDataset for multi-label classification with configurable max_length=256 tokenization.
  • Loads labeled data in JSONL format with enum_labels field mapping to the 64-category taxonomy.
  • Outputs model artifacts to model.tar.gz for endpoint deployment via deploy_endpoint.py.

Results

  • Production-deployed pipeline processing alerts 24/7 with automatic SQS-driven scaling.
  • Hybrid architecture reduced Bedrock API costs by routing high-confidence cases through SageMaker first.
  • Chain-of-thought prompting improved accuracy on ambiguous categories (e.g., distinguishing ENVIRONMENT_WILDFIRE from CRIME_ARSON).
  • Category validation caught 12% of misclassifications in production before downstream routing.

Limitations

  • Lambda cold starts add 2-5 second latency on first invocation after idle periods—mitigated with provisioned concurrency for critical workloads.
  • SageMaker endpoint adds fixed hourly cost; hybrid mode balances this against per-request Bedrock pricing.
  • Multi-label classification struggles with extremely rare categories (<1% of training data) due to class imbalance.
